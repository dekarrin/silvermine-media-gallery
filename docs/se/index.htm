Coppermine Photo Gallery v1.5.x: Dokumentation och Manual

Innehållsförteckning

Installation och Setup

Hur man installerar programmet

  • Minimikrav

    Se till att du uppfyller minimikraven för att köra Coppermine på din webbserver.

  • Packa upp arkivet och behåll katalogstrukturen

    Du kan (och bör) döpa om coppermine mappen, men inte filerna och mapparna i den.

    Detaljer

    Du behöver en programvara som kan packa upp arkivet, t.ex. 7-Zip. Om inte ditt OS stöder uppackning (Windows XP eller högre har support för zip filer; de flesta Linux distributioner kommer också med motsvarande som File Roller). I en del packning/Uppackningsprogram som WinZip finns det en möjlighet för användaren att välja om katalogstrukturen inom arkivet ska bibehållas intakt eller om alla filer som arkivet innehåller ska packas upp till en mapp utan undermappar. För Coppermine måste du välja att bibehålla katalogstrukturen.

    Ett bra ställe att packa upp din kopia av Coppermine är Skrivbordet eller en temporär mapp. I den här beskrivningen förutsätter vi att du packat upp arkivet till skrivbordet.

    Efter att du packat upp, bör du ha en mapp på ditt skrivbord som heter cpg15x. Du kan (och bör) ändra namnet till något vettigare för användning på internet, men säkerställ att du använder ett namn som är webbsäkert: använd inte specialtecken utom bindestreck (-) och understrykningstecken (_), använd inte mellanslag, tecken med apostrof eller andra specialtecken. Undvik versaler om möjligt. I själva verket bör du endast använda latinska alfanumeriska tecken (a till z och 0 till 9). Denna rekommendation gäller inte bara Coppermine mappen, utan alla mappar och filer som du laddar upp till webb-servern.

    Här är en förteckning på bra namn på mappen:
    • galleri
    • bilder
    • foton
    • multimedia
    Det namn som du väljer kommer senare att bli sökväg till ditt galleri, så det gäller att välja ett namn som är lätt att komma ihåg. Du får en liten extrabonus vad gäller sökmotorvänlighet om du väljer ett beskrivande namn som galleri. Bekymra dig inte om vilket namn du väljer för den mapp som Coppermine kommer läggas i: du kan alltid ändra namn på mappen senare.

  • Ladda upp alla filer till din webbserver

    Det är obligatoriskt att ladda upp alla mappar och filer som ingår i Coppermine paketet till din webbserver.
    Detaljer

    Använd din FTP programvara för att ladda upp mappen som du packade upp Coppermine i till din webbserver. Använd inte tilläggskomponenter till webbläsaren eller andra stöd för att ladda upp filerna, utan använd ett riktigt FTP program. Använd inte uppladdningmetoder som kommer med WYSIWYG-editorer som MS Frontpage eller Dreamweaver, eftersom dessa uppladdare har en tendens att försköna kodfilerna, och skadar dem på så sätt. Dessa editorer och de verktyg som de innehåller är OK att använda för andra syften, men de skapar ofta problem för färdiggjorda scripts som Coppermine, därför rekommenderar Coppermines utvecklingsteam starkt att inte använda sådana WYSIWYG-editorer alls. Om du inte har någon FTP applikation ännu, se förteckningen på verktyg som rekommenderas av Coppermines utvecklare.

    För att kunna använda din FTP applikation, måste du förstås veta en del FTP detaljer om din hemsida. Om du inte är säker på dessa detaljer, fråga supporten till ditt webbhotell.

    Kontrollera att du använder rätt FTP mode när du laddar upp.

    Om du tänker använda Coppermine som din primära applikation som visas på första sidan på din hemsida, så måste du inte nödvändigtvis lägga Coppermines filer i en separat mapp: då kan du tryggt ladda upp innehållet i Coppermine paketet direkt på roten på din server.

  • Sätt behörigheter på "albums" och "include" mapparna i din Coppermine mapp

    Ange behörigheter på filsystemsnivå för Coppermine scriptet. Coppermine behöver skrivbehörigheter för en del undermappar i Coppermine mappen. Att sätta behörigheter kan ibland vara svårt för den som är ovan, därför finns ett separat avsnitt i dokumentationen som förklarar detta i detalj.
    Vanligtvis måste du använda kommandot CHMOD för att sätta behörigheter till 755 (eller 777, beroende på din serverkonfiguration).
    Detta steg är mycket viktigt och får inte förbises!

  • Kontrollera att du har rätt uppgifter om din databas

    Du måste veta namnet på databasen så väl som detaljer om det MySQL användarkonto som Coppermine behöver använda för att ansluta till databasen. Databasen och användarnamnet måste redan finnas, och användaren måste ha behörighet till relevant databas. Coppermine kommer inte att skapa databasen åt dig, men det kommer att skapa tabeller i databasen under installationen, du behöver inte lägga till tabellerna själv.
    Detaljer
    Om det inte finns någon databas för din hemsida, måste du skapa en eller få supporten för ditt webbhotell att skapa en. Ditt webbhotell är de som är bäst lämpade att ge dig instruktioner om hur du ska göra för att åstadkomma detta. För installationen behöver du veta sökvägen till den databasserver som du kommer att använda med Coppermine och detaljer om ditt MySQL användarkonto som Coppermine kommer att ansluta till servern med. Du måste ha sökvägen till databasservern (vanligtvis 'localhost'), ditt MySQL användarnamn, MySQL lösenord, och MySQL databasnamn före installationen av Coppermine. Användaren som installerar Coppermine måste ha alla relevanta behörigheter till denna databas.
    Om du använder en nyligen skapad databas, finns det ingen anledning för dig att manuellt lägga till några tabeller. Coppermine kommer att skapa de nödvändiga tabellerna i databasen under installation.

  • Kör installationen på din server

    Skriv följande URL i adressfältet i webbläsaren: http://din_server/coppermine_dir/install.php (din_server = din hemsida, coppermine_dir = den mapp dit du laddade upp dina Coppermine filer.) Följ instruktionerna på installationsskärmen och ange på begäran nödvändiga uppgifter.
    Exempel
    Om din hemsida heter exempel.se och namnet på din Coppermine mapp är galleri, så är adressen du ska skriva in http://exempel.se/galleri/install.php

Vad installationsprogrammet gör

Efter att ha utfört vissa grundläggande kontroller skapar installationsprogrammet de nödvändiga databastabellerna år dig och fyller dem med standardvärden. Den skapar filen include/config.inc.php i Coppermine mappen på den server som innehåller de databasdetaljer som du angav under installationen. Om du skulle ändra dina mysql databasdetaljer senare (t.ex. om du ändrar lösenord på ditt mysql användarkonto eller om du flyttar ditt galleri till en annan server), kommer du att behöva redigera include/config.inc.php manuellt för att återspegla förändringarna. Filen include/config.inc.php förhindrar också att installationsprogrammet körs två gånger: om installationsprogrammet körs, kontrollerar det om config filen redan finns - och om det gör det stannar programmet och omdirigerar användaren till index sidan.

Server konfigurering

När du installerar Coppermine, kan det hända att du får problem som inte är direkt relaterat till, utan med hur webbserver är uppsatt. Därför kan det här avsnittet av dokumentationen bara ge dig ungefärlig överblick över problem med webbserver inställningar. Vi kan bara ge rekommendationer, men det är inte i första hand Coppermine supportens sak att faktiskt ge råd om hur en webbserver ska konfigureras. Alltså: om du har problem med webbserverinställningen, be supporten för ditt webbhotell om hjälp istället för att ställa frågor till Coppermine supporten.

register_globals

I äldre version av PHP, är standardinställningen för register_globals "on", vilket fick till följd att många PHP kodare skrev säkerhetsmässigt slarvig kod (helt enkelt för att de inte var medvetna om farorna). Som ett resultat av det brukade det finnas många script som var beroende av att server variabeln "register_globals" var satt på. Eftersom många ondsinta attacker mot script utförs som är beroende av att register_globals är på, är till tillrådigt att sätta av den i PHP inställningarna på din webbserver. I nyare versioner av PHP, är standardinställningen "off". Ditt webbhotell kan emellertid ändå ha aktiverat register_globals för att göra det möjligt att köra gamla script som är beroende av att register_globals är på.

För att göra detta fullständigt klart: Coppermine behöver inte ha register_globals=on, utan kan köras med både register_globals on eller off. Det är bara säkerhetsaspekten av att denna variabel är på som vi vill göra användaren uppmärksam på. Om din webbserver har register_globals satt på, kommer installationsprogrammet att varna dig om detta, men det kommer att fortsätta ändå.
