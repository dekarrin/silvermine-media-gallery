Kända problem

Följande små egenheter och små problem är kända och kommer förhoppningsvis as rättas i någon senare release.

  • De pilar som används för navigering från en bild till en annan visas i fel ordning för rtl-språk (höger-till-vänster)
  • Nyckelordslistan visar fel resultat för asiatiska tecken
  • Webbläsar bug: IE visar 100%-breddvärden inkorrekt
  • Nya batch-uppladdningsgränssnitt fungerar inte för alla serverinställningar. Användare som har problem med det bör använda klassiskt gränssnitt istället.
  • Sidflikar i användarhanteraren fungerar inte på samma sätt som andra flikar
  • Skalning av gif/png med transparent bakgrund bevarar inte alfabetisk information
  • Fel teckenuppsättning i e-post som skickas från Coppermine
  • Miniatyrbildsstorlekar över 120px orsakar att redigeringsfält flödare över kanterna
  • Inga miniatyrbilder för album som bara innehåller bilder länkade med nyckelord
  • Beskär/rotera fungerar inte med GD1
  • Rotering orsakar att stora bilder förvrängs
  • <object><param></param></object> konstruktioner för multimedia filer valideras inte men behöver för kompatibilitet bakåt
  • EXIF och IPTC information försvinner vid rotering
  • Skillnader mellan Coppermine som internt använder utf-8 och IPTC som för närvarande stöder iso-8859-1. Därför blir icke latinska tecken som t.ex. tecken med apostrof förvrängda.
  • Album skapas vid användning av XP publisher trots att uppladdningen misslyckas
  • Ingen kategoriminiatyrbild för användargalleriers och kategorier som inte innehåller album med bilder, utan endast subkategorier
  • Filmremsor kan förändra sin höjd när navigering i miniatyrdimensioner är satt till 'bredd'
  • Flash uppladdare (swfupload) fungerar inte om galleriet finns i lösenordsskyddade bibliotek
  • En del effekter med rundade hörn i curve-temat fungerar inte för alla webbläsare utan bara vissa. Dom fungerar inte i IE huvudsakligen. Webbläsare som inte kan visa rundande hörn kommer att visa "normala" hörn.
  • Album moderator funktionen inte tillräckligt tydligt dokumenterad och därför inaktiverad (thread)
  • Kategoriminiatyrbilder visas större än förväntat om miniatyrinställningen är satt till "exakt" "Använd dimension" (thread)
  • Om du tillåter gäster att ladda upp filer till ditt galleri, kommer de inte att kunna ange fildetaljer. Denna funktion har inaktiverats eftersom det inte är möjligt att felfritt identifiera användaren (se thread). Dessutom kan det locka spammare att lägga till textinnehåll till ditt galleri. Om du vill aktivera den här funktionen ändå, hitta Coppermines konfigurationstabell i databasen och ändra värdet på allow_guests_enter_file_details till '1'.
